# Menu

A dropdown or context menu that anchors to any target and just works. Place
`<Menu>` immediately after the trigger вҖ” a button, an icon, a whole region вҖ”
and it opens on click, positions itself with room-aware flipping, and dismisses
on outside-click, `Escape`, or scroll. Rows are dedicated **`MenuItem`s** (not
buttons), with optional leading icons, trailing keyboard hints, separators,
grouped sections, and nested submenus.

Open state is uncontrolled: listen for the `open` / `close` events or grab a
`ref` to the `<a-menu>` element and call `.open()` / `.close()` / `.toggle()`.

The previews below are live вҖ” **click a trigger to open its menu**. Play with
the full API in the [playground](#playground).

## Dropdown from a button

<Columns>
<Col>

Render `<Menu>` right after the trigger element. A `MenuItem` takes a leading
`icon`, a `label` (or `children`), and a trailing `kbd` hint. Selecting an item
closes the menu and fires its `onSelect`; `Escape` or an outside click dismisses
without selecting.

</Col>
<Col>

<Preview>

  <Menu>
    <MenuItem icon="edit" label="Edit" kbd="вҢҳE" />
    <MenuItem icon="copy" label="Duplicate" kbd="вҢҳD" />
    <MenuItem icon="download" label="Export" />
    <MenuSeparator />
    <MenuItem tone="critical" icon="trash" label="Delete" kbd="вҢҳвҢ«" />
  </Menu>
</Preview>

</Col>

## Tones

<Columns>
<Col>

`tone` colors an item's label, icon, and hover tint with a semantic palette вҖ”
`brand`, `info`, `success`, `warning`, or `critical` вҖ” tracking light and dark
mode automatically. Omit it (or pass `neutral`) for the standard gray. `critical`
is the conventional destructive action.

</Col>
<Col>

## Placement

<Columns>
<Col>

`placement` sets the preferred side and edge alignment: `bottom-start` (the
default), `bottom-end`, `top-start`, or `top-end`. The `-start` / `-end` suffix
aligns the menu's left or right edge to the trigger. The menu auto-flips
vertically and clamps horizontally when there isn't room, so the placement is a
preference, not a constraint.

</Col>
<Col>

<Preview>
  <Button>bottom-start</Button>
  <Menu placement="bottom-start">
    <MenuItem label="One" />
    <MenuItem label="Two" />
  </Menu>
  <Button>bottom-end</Button>
  <Menu placement="bottom-end">
    <MenuItem label="One" />
    <MenuItem label="Two" />
  </Menu>
</Preview>

## Right-click & at-cursor

<Columns>
<Col>

Pass `context` to open on right-click (the `contextmenu` event) of the region
the menu follows, positioned at the pointer вҖ” ideal for content surfaces like a
log viewer or canvas where secondary actions shouldn't clutter the UI. `coord`
on its own opens a normal left-click menu at the cursor instead of aligned to
the trigger box.

</Col>
<Col>

    <MenuItem icon="link" label="Share log" />
    <MenuItem icon="copy" label="Copy moment" />
  </Menu>
</Preview>

## Submenus

<Columns>
<Col>

Mark a `MenuItem` with `submenu` and nest a `<Menu submenu>` inside it to create
a flyout. The parent item gets a chevron and `aria-haspopup`; the submenu opens
to the side and flips when it nears the edge. Submenus open on click by default вҖ”
add `hover` to also open them on hover (with intent timing, so a quick
pass-through doesn't trigger them).

</Col>
<Col>

## Groups

<Columns>
<Col>

Wrap related items in a `MenuGroup` with a `label` to add a titled section.
Headings are skipped during keyboard navigation, which flattens all items into a
single up/down sequence across group boundaries.

## Custom content & the close contract

<Columns>
<Col>

Selecting a `MenuItem` closes the menu. **Arbitrary content you inject is never
treated as an item, so it never closes the menu** вҖ” drop a slider, an input, or
any control straight into a `<Menu>` and interacting with it leaves the menu
open. To keep the menu open after an actual item is chosen (a toggle, say), set
`keepOpen` on the item or on a wrapping `MenuGroup`. For imperative control from
a custom handler, call `.close()` on the `<a-menu>` element via a `ref`.

</Col>
<Col>

<Preview>
  <Button>Settings</Button>
  <Menu>
    <MenuItem label="Profile" />
    <MenuItem label="Notifications" keepOpen />
    <MenuSeparator />
    <div style="padding:8px;display:flex;flex-direction:column;gap:4px;min-width:180px">
      <label style="font-size:12px;color:var(--text-4)">Zoom</label>
      <input type="range" />
    </div>
  </Menu>
</Preview>

```tsx folded
<Menu>
  <MenuItem label="Profile" onSelect={openProfile} />
  {/* keepOpen: a toggle that doesn't dismiss */}
  <MenuItem label="Notifications" keepOpen onSelect={toggleNotifs} />
  <MenuSeparator />
  {/* custom content вҖ” interacting with it never closes the menu */}
  <div class="zoom"><label>Zoom</label><input type="range" /></div>
</Menu>
```

</Col>
</Columns>

## Keyboard

The menu manages focus without trapping it. Open the trigger with `Enter` /
`Space` to focus the first item.

- **вҶ‘ / вҶ“** вҖ” move between items (wraps; skips separators, headings, and disabled items)
- **Home / End** вҖ” first / last item
- **Type-ahead** вҖ” type to jump to the next item whose label matches
- **вҶ’ / Enter** вҖ” open a submenu (and focus its first item); **вҶҗ** вҖ” close it, back to the parent
- **Enter / Space** вҖ” activate the focused item
- **Esc** вҖ” close the topmost menu and return focus to its parent item or trigger
- **Tab** вҖ” close the menu and continue tabbing through the page

<Disclosure title="Component tokens">

These tokens theme the **surface** вҖ” the popover chrome that lives in shadow DOM
and so can't be reached with plain CSS. Override them on a single menu
(`style={{ '--menu-bg': 'var(--bg-2)' }}`) or on a selector wrapping it.

| Token | Description |
|---|---|
| `--menu-bg` | Surface background fill (frosted via `--menu-backdrop-filter`). |
| `--menu-border` | Surface hairline border. |
| `--menu-radius` | Surface corner radius. |
| `--menu-shadow` | Surface drop shadow (lift). |
| `--menu-padding` | Inner padding around the items. |
| `--menu-min-width` | Minimum surface width. |
| `--menu-backdrop-filter` | Backdrop blur behind the surface. |

**Menu items aren't tokenized.** `a-menu-item` is light DOM, so style it вҖ” and
its `a-icon` / `kbd` / `a-menu-item-label` children вҖ” directly with your own CSS
(e.g. `a-menu-item { font-size: 13px }` or a class). The `tone` attribute already
covers the semantic palette.

</Disclosure>
