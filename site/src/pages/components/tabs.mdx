# Tabs

A tablist for switching between views. Compose it from `<Tab>` children ŌĆö each
takes a unique `value` and a `label` (or `children`), plus optional `icon` /
`iconTrailing`. That alone is a selectable strip that emits the chosen value; add
matching `<TabPanel>`s (see [Panels](#panels)) when you want `Tabs` to show and
hide the content too.

strip's labels, indicator, and track at every priority. Rows are tones (the last is a

Pair a `<TabPanel value="ŌĆ”">` to the `<Tab>` of the same value and `Tabs` shows

Panels are **optional**. With none, `Tabs` is just the strip: listen to
`onStateChange` / `onAnyChange` and render whatever you like yourself.

<Disclosure title="What it renders (JSX vs. DOM)">

`<Tab>` and `<TabPanel>` are **config-only** components ŌĆö they render nothing
themselves; `Tabs` reads their props. The flat, interleaved JSX above splits into a
strip element with the panels as its **siblings** (not children), inside a layout
wrapper:

```html
<div>                              <!-- layout wrapper, only when panels exist -->
  <a-tabs role="tablist">
    <a-tab role="tab" id="ŌĆ”-tab-account" aria-controls="ŌĆ”-panel-account" tabindex="0">
      <a-tab-label>Account</a-tab-label>
    </a-tab>
    ŌĆ”
  </a-tabs>
  <a-tabpanel role="tabpanel" id="ŌĆ”-panel-account" aria-labelledby="ŌĆ”-tab-account">ŌĆ”</a-tabpanel>
  <a-tabpanel ŌĆ” hidden inert>ŌĆ”</a-tabpanel>   <!-- inactive: hidden/unmounted per `mounting` -->
</div>
```

Each `<TabPanel>` becomes an `<a-tabpanel>` **beside** `<a-tabs>`, never inside it ŌĆö so
a panel never sits in the primary track (the track background lives on `<a-tabs>`
itself). The tabŌåöpanel link is by `id` (`aria-controls` / `aria-labelledby`), so their
adjacency isn't required for correctness.

</Disclosure>

### Mounting

<Columns>
<Col>

```tsx folded
{/* default ŌĆö every panel mounted, inactive ones display:none */}
<Tabs defaultValue="a">
  <Tab value="a" label="A" /><Tab value="b" label="B" />
  <TabPanel value="a"><Expensive /></TabPanel>
  <TabPanel value="b"><Form /></TabPanel>
</Tabs>

{/* render only the active panel; React unmounts the rest */}
<Tabs mounting="active" defaultValue="a">ŌĆ”</Tabs>

{/* per-panel override */}
<TabPanel value="b" mounting="lazy"><Heavy /></TabPanel>
```

</Col>
<Col>

`mounting` controls what happens to **inactive** panels. Set it on `Tabs` for all
panels, or on a single `<TabPanel>` to override one.

- **`display`** (default) ŌĆö every panel is mounted; inactive ones hide via
  `display:none`. Preserves their DOM/form state, and a hidden panel correctly
  leaves the tab order + a11y tree.
- **`visibility`** ŌĆö same, but hidden via `visibility:hidden`, so each panel keeps
  its layout box (useful to avoid reflow or to measure).
- **`active`** ŌĆö only the active panel is rendered; React unmounts the others
  (resets their state on switch; defers expensive subtrees).
- **`lazy`** ŌĆö a panel mounts on first activation, then stays mounted+hidden.

</Col>
</Columns>

## Keyboard & accessibility

Every enabled tab is in the tab order ŌĆö `Tab` / `Shift`+`Tab` step through them.
The arrow keys also move between enabled tabs (wrapping at the ends), `Home` /
`End` jump to the first / last, and `Space` / `Enter` activate the focused tab;
arrow / `Home` / `End` navigation activates as it moves (selection follows focus).
Disabled tabs are skipped and dropped from the tab order.

Pass `label` for the tablist's accessible name. Each `<TabPanel>` is labelled by
its tab (`aria-labelledby`) and pointed at from it (`aria-controls`).

## Controlled vs. uncontrolled

<Columns>
<Col>

**Uncontrolled** ŌĆö pass `defaultValue` and let `Tabs` own the active tab:

```tsx folded
<Tabs defaultValue="overview">
  <Tab value="overview" label="Overview" />
  <Tab value="activity" label="Activity" />
</Tabs>
```

</Col>
<Col>

**Controlled** ŌĆö pass `value` + `onStateChange` and own it yourself. The strip
fires *before* applying, so accept by updating `value` and reject by doing nothing
(or `event.preventDefault()` to veto in uncontrolled mode):

```tsx folded
const [tab, setTab] = useState('overview')

<Tabs value={tab} onStateChange={(e, { next }) => setTab(next)}>
  <Tab value="overview" label="Overview" />
  <Tab value="activity" label="Activity" />
</Tabs>
```

</Col>
</Columns>

## Events

Three callbacks, in firing order:

| Callback | When | Cancelable | Payload |
|---|---|---|---|
| `onStateChange(e, { next, prev })` | **before** a pick applies | Ō£ģ | next / prev value |
| `onChange(e)` | **after** selection applies | ŌĆö | native `change` event |
| `onAnyChange(e, { value })` | **after** selection applies | ŌĆö | `{ value }` |

Reach for **`onStateChange`** to intercept or veto a pick; for the everyday "react
to the new value", use **`onAnyChange`**. **`onFocus` / `onBlur`** report focus
entering and leaving the *strip* (wired to `focusin` / `focusout`, since focus
lands on an individual tab).

```tsx folded
<Tabs onAnyChange={(_e, { value }) => route(value)}>ŌĆ”</Tabs>
```
