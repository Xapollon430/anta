# Tabs

A tablist for switching between views. Compose it from `<Tab>` children — each
takes a unique `value` and a `label` (or `children`), plus optional `icon` /
`iconTrailing`. That alone is a selectable strip that emits the chosen value; add
matching `<TabPanel>`s (see [Panels](#panels)) when you want `Tabs` to show and
hide the content too.

## Tones

Six named tones — `neutral` (default), `brand`, `info`, `success`, `warning`,
`critical` — plus any CSS colour for a one-off custom tone. The tone runs through the
strip's labels, indicator, and track at every priority. Rows are tones (the last is a
custom colour); columns are priorities.

## Orientation

`horizontal` (default) lays the tabs in a row; `vertical` stacks them, and a paired
panel sits beside the strip. The arrow keys follow the axis — `←`/`→` when horizontal,
`↑`/`↓` when vertical. Vertical works at every priority:

## Panels

<Columns>
<Col>

<Preview>
  <Basic client:only="preact" />
  <style is:inline>{`
    .panels-demo > a-tabs { align-self: center; }
  `}</style>
</Preview>

```tsx folded
<Tabs defaultValue="account" label="Settings">
  <Tab value="account" label="Account" icon="home" />
  <Tab value="security" label="Security" />
  <Tab value="billing" label="Billing" />

  <TabPanel value="account"><AccountForm /></TabPanel>
  <TabPanel value="security"><SecurityForm /></TabPanel>
  <TabPanel value="billing"><BillingForm /></TabPanel>
</Tabs>
```

</Col>
<Col>

Pair a `<TabPanel value="…">` to the `<Tab>` of the same value and `Tabs` shows
the active panel, wiring up the `aria-controls` / `aria-labelledby` link between
them. Panel content is arbitrary React — forms, charts, whatever — so it renders
normally.

Panels are **optional**. With none, `Tabs` is just the strip: listen to
`onStateChange` / `onAnyChange` and render whatever you like yourself.

</Col>
</Columns>

<Disclosure title="JSX vs DOM" level={3}>

`<Tab>` and `<TabPanel>` are **config-only** components — they render nothing
themselves; `Tabs` reads their props. The flat, interleaved JSX above splits into a
strip element with the panels as its **siblings** (not children), inside a layout
wrapper:

```html
<div>                              <!-- layout wrapper, only when panels exist -->
  <a-tabs role="tablist">
    <a-tab role="tab" id="…-tab-account" aria-controls="…-panel-account" tabindex="0">
      <a-tab-label>Account</a-tab-label>
    </a-tab>
    …
  </a-tabs>
  <a-tabpanel role="tabpanel" id="…-panel-account" aria-labelledby="…-tab-account">…</a-tabpanel>
  <a-tabpanel … hidden inert>…</a-tabpanel>   <!-- inactive: hidden/unmounted per `mounting` -->
</div>
```

Each `<TabPanel>` becomes an `<a-tabpanel>` **beside** `<a-tabs>`, never inside it — so
a panel never sits in the primary track (the track background lives on `<a-tabs>`
itself). The tab↔panel link is by `id` (`aria-controls` / `aria-labelledby`), so their
adjacency isn't required for correctness.

</Disclosure>

### Mounting

<Columns>
<Col>

```tsx folded
{/* default — every panel mounted, inactive ones display:none */}
<Tabs defaultValue="a">
  <Tab value="a" label="A" /><Tab value="b" label="B" />
  <TabPanel value="a"><Expensive /></TabPanel>
  <TabPanel value="b"><Form /></TabPanel>
</Tabs>

{/* render only the active panel; React unmounts the rest */}
<Tabs mounting="active" defaultValue="a">…</Tabs>

{/* per-panel override */}
<TabPanel value="b" mounting="lazy"><Heavy /></TabPanel>
```

</Col>
<Col>

`mounting` controls what happens to **inactive** panels. Set it on `Tabs` for all
panels, or on a single `<TabPanel>` to override one.

- **`display`** (default) — every panel is mounted; inactive ones hide via
  `display:none`. Preserves their DOM/form state, and a hidden panel correctly
  leaves the tab order + a11y tree.
- **`visibility`** — same, but hidden via `visibility:hidden`, so each panel keeps
  its layout box (useful to avoid reflow or to measure).
- **`active`** — only the active panel is rendered; React unmounts the others
  (resets their state on switch; defers expensive subtrees).
- **`lazy`** — a panel mounts on first activation, then stays mounted+hidden.

</Col>
</Columns>

## Keyboard & accessibility

Every enabled tab is in the tab order — `Tab` / `Shift`+`Tab` step through them.
The arrow keys also move between enabled tabs (wrapping at the ends), `Home` /
`End` jump to the first / last, and `Space` / `Enter` activate the focused tab;
arrow / `Home` / `End` navigation activates as it moves (selection follows focus).
Disabled tabs are skipped and dropped from the tab order.

Pass `label` for the tablist's accessible name. Each `<TabPanel>` is labelled by
its tab (`aria-labelledby`) and pointed at from it (`aria-controls`).

## Controlled vs. uncontrolled

<Columns>
<Col>

**Uncontrolled** — pass `defaultValue` and let `Tabs` own the active tab:

```tsx folded
<Tabs defaultValue="overview">
  <Tab value="overview" label="Overview" />
  <Tab value="activity" label="Activity" />
</Tabs>
```

</Col>
<Col>

**Controlled** — pass `value` + `onStateChange` and own it yourself. The strip
fires *before* applying, so accept by updating `value` and reject by doing nothing
(or `event.preventDefault()` to veto in uncontrolled mode):

```tsx folded
const [tab, setTab] = useState('overview')

<Tabs value={tab} onStateChange={(e, { next }) => setTab(next)}>
  <Tab value="overview" label="Overview" />
  <Tab value="activity" label="Activity" />
</Tabs>
```

</Col>
</Columns>

## Events

Three callbacks, in firing order:

| Callback | When | Cancelable | Payload |
|---|---|---|---|
| `onStateChange(e, { next, prev })` | **before** a pick applies | ✅ | next / prev value |
| `onChange(e)` | **after** selection applies | — | native `change` event |
| `onAnyChange(e, { value })` | **after** selection applies | — | `{ value }` |

Reach for **`onStateChange`** to intercept or veto a pick; for the everyday "react
to the new value", use **`onAnyChange`**. **`onFocus` / `onBlur`** report focus
entering and leaving the *strip* (wired to `focusin` / `focusout`, since focus
lands on an individual tab).

```tsx folded
<Tabs onAnyChange={(_e, { value }) => route(value)}>…</Tabs>
```

<Disclosure title="Styling">

Reach for the props first: **`tone`**, **`size`**, **`priority`**. The focus ring is
the global [`--focus-ring`](/colors/#focus-ring).

For anything else the strip is light-DOM — `<a-tabs>` the tablist, each `<a-tab>` a
tab (its text wrapped in `<a-tab-label>`), `<a-tabpanel>` a panel, and the moving
indicator is `<a-tabs>`'s `::before`. Target them with ordinary CSS — an un-layered rule
beats `@layer anta` without `!important` — and reach for real properties
(`border-radius`, `padding`, `background`, …), **not** the internal `--*` output tokens.
The demo classes below (`.square-tabs`, `.glow-tabs`, …) are just hooks; swap your own.

**Squarer primary** — square corners, heavier labels, a roomier track, and one
`--edge` colour (via `light-dark()`) shared by the track's 1px border and the pill's
crisp 1px ring (spread, no blur). The track is `<a-tabs>` and the sliding pill its
`::before`:

<Preview>
  <SquareTabs client:only="preact" />
  <style is:inline>{`
    a-tabs.square-tabs { --edge: light-dark(#cfcfd6, #46464e); border: 1px solid var(--edge); border-radius: 0; padding: 4px; }
    .square-tabs a-tab { border-radius: 0; font-weight: 600; }
    a-tabs.square-tabs::before { border-radius: 0; box-shadow: 0 0 0 1px var(--edge); }
  `}</style>
</Preview>

```css folded
/* one --edge colour drives the track border AND the pill's 1px ring */
a-tabs.square-tabs         { --edge: light-dark(#cfcfd6, #46464e); border: 1px solid var(--edge); border-radius: 0; padding: 4px; }
.square-tabs a-tab         { border-radius: 0; font-weight: 600; }
a-tabs.square-tabs::before { border-radius: 0; box-shadow: 0 0 0 1px var(--edge); }  /* 1px spread, no blur */
```

**Tertiary accent + highlight** — a thicker, recoloured underline on the selected tab
(1px at rest → 2px active) plus a faint radial highlight in the same accent rising from
the bottom. One `--accent` (`light-dark()`) drives both the border and the gradient. The
active tab trims 1px of bottom padding to offset the 2px border so labels stay aligned:

<Preview>
  <TertiaryGlow client:only="preact" />

    .glow-tabs a-tab { border-bottom-width: 1px; }
    .glow-tabs a-tab:state(selected) {
      --accent: light-dark(#5f4bc3, #a78bfa);
      border-bottom: 2px solid var(--accent);
      padding-bottom: 5px;
      background: radial-gradient(80% 100% at 50% 100%, color-mix(in oklch, var(--accent) 10%, transparent), transparent 70%);
    }
  `}</style>
</Preview>

```css folded
.glow-tabs a-tab { border-bottom-width: 1px; }              /* rest: 1px rule */
.glow-tabs a-tab:state(selected) {
  --accent: light-dark(#5f4bc3, #a78bfa);                   /* accent colour, per theme */
  border-bottom: 2px solid var(--accent);                   /* active: 2px accent underline */
  padding-bottom: 5px;                                      /* −1px, offsets the +1px border */
  background: radial-gradient(80% 100% at 50% 100%,         /* highlight rising from the bottom */
              color-mix(in oklch, var(--accent) 10%, transparent), transparent 70%);
}
```
