# Expander

A collapsible disclosure: a header that toggles a region of content open
and closed. The `Expander` JSX wrapper renders an `<a-expander>`, whose
shadow DOM is a `<button>` summary plus a collapsible region — so focus,
keyboard activation (<kbd>Enter</kbd> / <kbd>Space</kbd>), and the
WAI-ARIA disclosure semantics (`aria-expanded`) come from the platform
button. The expand / collapse animation matches the disclosure sections
used across these docs.

The `title` is the always-visible header; the `children` are the
collapsible body.

Play with it in [playground](#playground).

## Priority

`priority` sets the surface, reusing `Button`/`Text`'s vocabulary.
`primary` is the most pronounced card; `secondary` (the default) is a
subtle filled box; `tertiary` is transparent — just the header row and
its chevron, for an inline disclosure.

<Preview direction="column" gap="8px">
  <Expander title="Primary" priority="primary">
    <Text>A more pronounced card (<code>--bg-4</code> + <code>--border-4</code>).</Text>
  </Expander>
  <Expander title="Secondary (default)">
    <Text>A subtle fill (<code>--bg-2</code>) with a faint border (<code>--border-5</code>).</Text>
  </Expander>
  <Expander title="Tertiary" priority="tertiary">
    <Text>Transparent — no background or border.</Text>
  </Expander>
</Preview>

```tsx folded
<Expander title="Primary" priority="primary">…</Expander>
<Expander title="Secondary (default)">…</Expander>      {/* secondary is the default */}
<Expander title="Tertiary" priority="tertiary">…</Expander>
```

## Tones

Pick a semantic `tone` to tint the header text and — on the filled
priorities (`secondary` / `primary`) — the surface too. The set matches
`Button` / `Tag`: `neutral` (the default) plus `brand`, `info`,
`success`, `warning`, and `critical`. Color comes from the theme tokens,
so every tone tracks light and dark mode automatically. Every tone is
shown below across all three priorities, so you can compare (rows are
tones, columns are priorities):

`tone` also accepts **any literal CSS color** for a one-off tint — the
hue is kept while lightness and chroma are pinned to the named-tone curve
(and re-tuned for dark mode), the same mechanism as `Button` / `Tag`:

## Marker

By default the chevron sits **inside** the header row. With
`priority="tertiary"`, set `marker="outside"` to hang it in the left
gutter (negative space) so the **title and body sit flush** with the
surrounding content — the same layout as the section headers in these
docs. The element zeroes its left border + padding for this variant so
the title starts at the element's edge.

<Preview direction="column" gap="8px">
  <Expander priority="tertiary" marker="outside" title="Flush header, chevron in the gutter" defaultOpen>
    <Text>The chevron hangs to the left of the title; the title and this body align with the surrounding content.</Text>
  </Expander>
</Preview>

```tsx folded
<Expander priority="tertiary" marker="outside" title="…">…</Expander>
```

Use it where the expander sits in a content column with room to its left
(a prose measure); in a full-width / edge-to-edge container the chevron
would overhang the edge.

## Title & level

`title` takes a string or any node. For a string, `level` (1–6) applies
the matching heading **type scale** — the same scale as `<Title>` —
without changing the document outline.

<Preview direction="column" gap="8px">
  <Expander title="A level-2 header" level={2}>
    <Text>The summary uses the level-2 heading type scale.</Text>
  </Expander>
  <Expander title="A level-4 header" level={4}>
    <Text>And this one the level-4 scale.</Text>
  </Expander>
</Preview>

```tsx folded
<Expander title="A level-2 header" level={2}>…</Expander>
<Expander title="A level-4 header" level={4}>…</Expander>
```

When you need real heading **semantics** — so the title lands in a
page's table of contents — pass a `<Title>` (or your own markup) as the
`title` instead of a string:

```tsx
<Expander title={<Title level={5}>A real heading node</Title>}>…</Expander>
```

## Open state

Leave it uncontrolled and pass `defaultOpen` for the initial state — the
element owns open/close from there. To drive it yourself, pass `open`
and handle `onToggle(open)` (controlled): the expander then only follows
the prop, and clicking the summary just *requests* a change through
`onToggle` — apply it to your state to accept, or ignore it to reject
(e.g. a section that needs confirmation before closing). If you pass
`open`, you own it.

```tsx
// uncontrolled
<Expander title="Details" defaultOpen>…</Expander>

// controlled
const [open, setOpen] = useState(false)
<Expander title="Details" open={open} onToggle={setOpen}>…</Expander>
```

<Disclosure title="Component tokens">

Override any of these on a single instance
(`style={{ '--expander-text': 'var(--text-1)' }}`) or on a selector
wrapping the expander. The named tones re-point `--expander-text` and the
filled surfaces from the theme palette, so light / dark track for free.

| Token | Description |
|---|---|
| `--expander-text` | Header + body text color (and the chevron, via `currentColor`). Tones re-point this. |
| `--expander-bg` | Host background. The secondary fill by default; the primary fill on `primary`; transparent on `tertiary`. |
| `--expander-border` | Host border color. Follows the same per-priority surface; transparent on `tertiary`. |
| `--expander-bg-secondary`<br />`--expander-border-secondary` | Secondary (default) surface — a subtle fill (`--bg-2`) + faint edge (`--border-5`), re-pointed per tone. |
| `--expander-bg-primary`<br />`--expander-border-primary` | Primary surface — a more pronounced card (`--bg-4` + `--border-4`), re-pointed per tone. |
| `--expander-summary-font-size`<br />`--expander-summary-line-height`<br />`--expander-summary-font-weight` | Summary (header) typography. Mirrors the `<Title>` type scale; `level` re-points size + line-height. |
| `--expander-tone-source` | The color a custom (non-named) `tone` derives from. Set inline by the wrapper; set it yourself to drive the derived palette from your own variable. |

</Disclosure>

<Disclosure title="Parts">

Most styling is covered by the tokens above and by the light-DOM
`<a-expander-summary>` / `<a-expander-details>` children, which you can
target with plain CSS. For what a variable can't express — interaction
states on the header, or the chevron — the shadow internals are exposed
as [`::part`](https://developer.mozilla.org/en-US/docs/Web/CSS/::part):

| Part | What it is |
|---|---|
| `summary` | The header `<button>`. Reach states CSS variables can't — `::part(summary):hover`, `:focus-visible`, or the chevron via `::part(summary)::before`. |
| `content` | The collapsible body region. |

```css
a-expander::part(summary):hover { background: var(--bg-3); }
a-expander::part(summary)::before { transform: scale(1.2); } /* the chevron */
```

</Disclosure>
