# Expander

A collapsible disclosure: a header that toggles a region of content open
and closed. The `Expander` JSX wrapper renders an `<a-expander>`, whose
shadow DOM is a `<button>` summary plus a collapsible region — so focus,
keyboard activation (<kbd>Enter</kbd> / <kbd>Space</kbd>), and the
WAI-ARIA disclosure semantics (`aria-expanded`) come from the platform
button. The expand / collapse animation matches the disclosure sections
used across these docs.

The `title` is the always-visible header; the `children` are the
collapsible body.

## Priority

`priority` sets the surface, reusing `Button`/`Text`'s vocabulary.
`primary` is the most pronounced card; `secondary` (the default) is a
subtle filled box; `tertiary` is transparent — just the header row and
its chevron, for an inline disclosure.

<Preview direction="column" gap="8px">
  <Expander title="Primary" priority="primary">
    <Text>A more pronounced card (<code>--bg-4</code> + <code>--border-4</code>).</Text>
  </Expander>
  <Expander title="Secondary (default)">
    <Text>A subtle fill (<code>--bg-2</code>) with a faint border (<code>--border-5</code>).</Text>
  </Expander>
  <Expander title="Tertiary" priority="tertiary">
    <Text>Transparent — no background or border.</Text>
  </Expander>
</Preview>

```tsx folded
<Expander title="Primary" priority="primary">…</Expander>
<Expander title="Secondary (default)">…</Expander>      {/* secondary is the default */}
<Expander title="Tertiary" priority="tertiary">…</Expander>
```

## Tones

Pick a semantic `tone` to tint the header text and — on the filled
priorities (`secondary` / `primary`) — the surface too. The set matches
`Button` / `Tag`: `neutral` (the default) plus `brand`, `info`,
`success`, `warning`, and `critical`. Color comes from the theme tokens,
so every tone tracks light and dark mode automatically. Every tone is
shown below across all three priorities, so you can compare (rows are
tones, columns are priorities):

## Outdent

By default the chevron sits **inside** the header row. On a transparent
`priority="tertiary"` expander, add `outdent` to hang the chevron in the
left gutter so the **title and body sit flush** with the surrounding
content — the same layout as the section headers in these docs. The
border stays (transparent, so the box geometry is unchanged); only the
left padding is zeroed, so the title sits at the element's edge.

<Preview direction="column" gap="8px">
  <Expander priority="tertiary" outdent title="Flush header, chevron in the gutter" defaultOpen>
    <Text>The chevron hangs to the left of the title; the title and this body align with the surrounding content.</Text>
  </Expander>
</Preview>

```tsx folded
<Expander priority="tertiary" outdent title="…">…</Expander>
```

`outdent` only takes effect on `tertiary` — on the filled priorities
(`secondary` / `primary`) the container edge has to bound the chevron, so
it's a no-op there. Use it where the expander sits in a content column
with room to its left (a prose measure); in a full-width / edge-to-edge
container the chevron would overhang the edge.

## Title & level

`title` takes a string or any node. For a string, `level` (1–6) applies
the matching heading **type scale** — the same scale as `<Title>` —
without changing the document outline. Only the type scale changes: the
title's left alignment is constant across every level (the chevron + a
fixed 4px edge inset), so swapping `level` never shifts the header
horizontally.

<Preview direction="column" gap="8px">
  <Expander title="Level 1" level={1}><Text>The level-1 type scale.</Text></Expander>
  <Expander title="Level 2" level={2}><Text>The level-2 type scale.</Text></Expander>
  <Expander title="Level 3" level={3}><Text>The level-3 type scale.</Text></Expander>
  <Expander title="Level 4" level={4}><Text>The level-4 type scale.</Text></Expander>
  <Expander title="Level 5 (default)"><Text>The compact default.</Text></Expander>
  <Expander title="Level 6" level={6}><Text>The smallest type scale.</Text></Expander>
</Preview>

```tsx folded
<Expander title="Level 1" level={1}>…</Expander>
<Expander title="Level 2" level={2}>…</Expander>
<Expander title="Level 3" level={3}>…</Expander>
<Expander title="Level 4" level={4}>…</Expander>
<Expander title="Level 5 (default)">…</Expander>      {/* level 5 is the default */}
<Expander title="Level 6" level={6}>…</Expander>
```

When you need real heading **semantics** — so the title lands in a
page's table of contents — pass a `<Title>` (or your own markup) as the
`title` instead of a string:

```tsx
<Expander title={<Title level={5}>A real heading node</Title>}>…</Expander>
```

## Actions

Pass `actions` to render extra controls — buttons, tags — at the end of
the header row. They sit **outside** the toggle trigger, as its flex
siblings: clicking them never toggles the expander, they're separately
focusable, and screen readers announce them as their own controls.
(Don't put interactive elements *inside* a custom `title` instead —
nesting controls in a button is an accessibility anti-pattern.)

<Preview direction="column" gap="8px">
  <Expander title="Build artifacts" actions={<Button size="small" label="Download" />}>
    <Text>The download button lives in the header but is not part of the trigger.</Text>
  </Expander>
</Preview>

```tsx folded
<Expander title="Build artifacts" actions={<Button size="small" label="Download" />}>
  …
</Expander>
```

The header never wraps: actions keep their intrinsic size and the
title ellipsizes first, so keep actions compact (a small button or
two, a tag). Pass several controls as a fragment — they form a row
with an 8px gap:

<Preview direction="column" gap="8px">
  <Expander
    title="A long section title that gives way to the actions and ellipsizes when space runs out"
    actions={<><Button size="small" label="Share" /><Button size="small" label="Download" /></>}
  >
    <Text>Actions hold their size; the title truncates.</Text>
  </Expander>
</Preview>

```tsx folded
<Expander
  title="A long section title that gives way to the actions…"
  actions={<><Button size="small" label="Share" /><Button size="small" label="Download" /></>}
>
  …
</Expander>
```

Actions stay visible in the folded state — the header represents the
section whether it's open or not (the same convention as GitHub's file
headers). If an action only makes sense while open, drive the expander
in controlled mode and swap the `actions` node on toggle.

### Composing the element directly

Without the JSX wrapper, the `slot` attribute is the wire format: put
`slot="title"` / `slot="actions"` directly on your own elements
(several elements can share `slot="actions"`); anything unslotted
becomes the collapsible body.

<Preview direction="column" gap="8px">
  <a-expander>
    <a-expander-summary slot="title">Hand-authored markup</a-expander-summary>
    <button slot="actions">Download</button>
    <a-expander-details>The body — everything without a <code>slot</code> attribute.</a-expander-details>
  </a-expander>
</Preview>

```html
<a-expander>
  <a-expander-summary slot="title">Hand-authored markup</a-expander-summary>
  <button slot="actions">Download</button>
  <a-expander-details>The body — everything without a slot attribute.</a-expander-details>
</a-expander>
```

## Disabled

`disabled` freezes the expander: the header is no longer clickable or
focusable, the hover affordance is off, and the text dims. The open
state stays as-is — disabling an open expander keeps it open. Header
`actions` stay live; disable them separately if needed.

<Preview direction="column" gap="8px">
  <Expander title="Disabled, closed" disabled>
    <Text>You can't see me.</Text>
  </Expander>
  <Expander title="Disabled, open" disabled defaultOpen>
    <Text>Content stays visible; the state is frozen.</Text>
  </Expander>
</Preview>

```tsx folded
<Expander title="Disabled, closed" disabled>…</Expander>
<Expander title="Disabled, open" disabled defaultOpen>…</Expander>
```

## Open state

Leave it uncontrolled and pass `defaultOpen` for the initial state — the
element owns open/close from there. To drive it yourself, pass `open`
and handle `onToggle(open)` (controlled): the expander then only follows
the prop, and clicking the summary just *requests* a change through
`onToggle` — apply it to your state to accept, or ignore it to reject
(e.g. a section that needs confirmation before closing). If you pass
`open`, you own it.

```tsx
// uncontrolled
<Expander title="Details" defaultOpen>…</Expander>

// controlled
const [open, setOpen] = useState(false)
<Expander title="Details" open={open} onToggle={setOpen}>…</Expander>
```

## Styling the chevron & header

There's deliberately no prop to hide the chevron: a region that folds
needs a visible affordance, so the default is to always show one. If you
have a genuine one-off need, the chevron is the summary's `::before`, so
you can remove it in CSS — zero the body indent too so the content stays
aligned with the now-unoffset title:

```css
a-expander::part(summary)::before { display: none; }
a-expander a-expander-details { padding-left: 0; }
```

To tint the **whole header** on hover / press, style the summary part
(the header `<button>`):

```css
a-expander::part(summary):hover  { background: var(--bg-3); }
a-expander::part(summary):active { background: var(--bg-2); }
```

Or just **highlight the title** — the header already eases the title to
`--expander-text-hover` on hover, so re-point that token (per tone if you
like) instead of writing your own hover rule:

```css
a-expander { --expander-text-hover: var(--text-1-brand); }
```

The header also exposes its open/closed state as the **`:state(open)`**
custom state, so your CSS can react to it (in controlled and uncontrolled
mode alike) — no JS. Two worked examples:

{/* Live CSS for the two examples below. `is:inline` so Astro renders it
    verbatim (the rules target web-component internals — ::part, :state — that
    a scoped style can't reach). The shown recipes use the same selectors. */}
<style is:inline>{`
  a-expander.code-like {
    --expander-text: var(--text-4);
    --expander-text-hover: var(--text-2);
    --expander-bg: var(--bg-canvas);
    --expander-border: var(--border-5);
  }
  a-expander.code-like::part(summary) { background: var(--bg-4); border-radius: 3px 3px 0 0; }

  a-expander.indicator::part(summary)::before { display: none; }
  a-expander.indicator a-expander-details { padding-left: 4px; }
  a-expander.indicator [slot="actions"] { pointer-events: none; color: var(--text-4); transition: transform 150ms ease; }
  a-expander.indicator:state(open) [slot="actions"] { transform: rotate(90deg); }
`}</style>

**Match a code block** — re-point the text + surface tokens and paint the
header bar with `::part(summary)`, so an Expander reads like the foldable
code blocks on this site:

<Preview direction="column" gap="8px">
  <Expander class="code-like" title="example.tsx" level={6} defaultOpen>
    <Text>The header reads like a code-block bar; the body is the editor surface.</Text>
  </Expander>
</Preview>

```css
a-expander.code-like {
  --expander-text: var(--text-4);
  --expander-text-hover: var(--text-2);
  --expander-bg: var(--bg-canvas);          /* body surface */
  --expander-border: var(--border-5);
}
a-expander.code-like::part(summary) { background: var(--bg-4); border-radius: 3px 3px 0 0; }
```

**Chevron as a status indicator** — drop the built-in toggle chevron and
put a **non-interactive** chevron in `actions` instead. The whole header
still toggles; the indicator only reflects state, rotating via
`:state(open)`:

<Preview direction="column" gap="8px">
  <a-expander class="indicator" defaultopen>
    <a-expander-summary slot="title">Build log</a-expander-summary>
    <a-icon slot="actions" shape="chevron-right"></a-icon>
    <a-expander-details><Text>The header is the trigger; the actions chevron is an indicator only.</Text></a-expander-details>
  </a-expander>
</Preview>

```html
<!-- the indicator goes in the actions slot; the whole header still toggles -->
<a-expander class="indicator">
  <a-expander-summary slot="title">Build log</a-expander-summary>
  <a-icon slot="actions" shape="chevron-right"></a-icon>
  <a-expander-details>…</a-expander-details>
</a-expander>
```
With the `<Expander>` JSX wrapper, pass `actions={<Icon shape="chevron-right" />}` instead; it wraps the icon in a layout-neutral span, so target `[slot="actions"] a-icon` in the rules below.

```css
/* drop the built-in toggle chevron + realign the body under the now-flush title */
a-expander.indicator::part(summary)::before { display: none; }
a-expander.indicator a-expander-details { padding-left: 4px; }
/* the actions chevron is an indicator only (never intercepts) and rotates with open */
a-expander.indicator [slot="actions"] { pointer-events: none; transition: transform 150ms ease; }
a-expander.indicator:state(open) [slot="actions"] { transform: rotate(90deg); }
```

<Disclosure title="Component tokens" open>

Override any of these on a single instance
(`style={{ '--expander-text': 'var(--text-1)' }}`) or on a wrapping
selector. These are the text + resolved-surface roles; tones re-point them
from the theme palette (so light / dark track for free) and `priority`
selects the fill. For interaction states beyond color — the header
hover/press background, the chevron — reach `::part(summary)` (see below).

| Token | Description |
|---|---|
| `--expander-text` | Header + body text color (and the chevron, via `currentColor`). |
| `--expander-text-hover` | Title color on header hover. Set it to recolor the hover highlight. (It's a token rather than a `:hover` rule because the slotted title takes the color via an inherited property — a Chromium slotted-invalidation workaround.) |
| `--expander-bg` | Host background — the resolved surface fill (`secondary` by default, `primary` on `priority="primary"`, transparent on `tertiary`). |
| `--expander-border` | Host border color — follows the same resolved surface. |
| `--expander-tone-source` | The color a custom (non-named) `tone` derives from. The wrapper sets it inline; set it yourself to derive the palette from your own variable. |

Each tone internally re-points per-priority pairs
(`--expander-{bg,border}-{secondary,primary}`) and `priority` picks which
pair is live — that's plumbing for the tone × priority matrix, not an
override surface. Reach for the resolved `--expander-bg` / `--expander-border`
(or `::part(summary)`) instead.

</Disclosure>

<Disclosure title="Parts">

Most styling is covered by the tokens above and by the light-DOM
`<a-expander-summary>` / `<a-expander-details>` children, which you can
target with plain CSS. For what a variable can't express — interaction
states on the header, or the chevron — the shadow internals are exposed
as [`::part`](https://developer.mozilla.org/en-US/docs/Web/CSS/::part):

| Part | What it is |
|---|---|
| `summary` | The header `<button>`. Reach what CSS variables can't — states (`::part(summary):hover`, `:focus-visible`), the chevron via `::part(summary)::before`, or bespoke header typography beyond what `level` offers. |
| `actions` | The header actions row (only rendered when actions are slotted) — e.g. to change its `gap` or alignment. |
| `content` | The collapsible body region. |

```css
a-expander::part(summary):hover { background: var(--bg-3); }
a-expander::part(summary)::before { transform: scale(1.2); } /* the chevron */
```

</Disclosure>
