# Button

An action control. Renders an `<a-button>` web component (or `<a role="button">`
in anchor mode) with a tone × priority matrix driven by attributes.

The playground opens on **All states** — every tone × priority × state combo
from the Figma component set (168 buttons). Switch examples in the Props
panel on the right.

## Tones and priorities

The button surface is a matrix of **6 tones** (`neutral`, `brand`, `critical`,
`info`, `success`, `warning`) crossed with **4 priorities**
(`primary`, `secondary`, `tertiary`, `quaternary`). The default is
`tone="brand" priority="primary"`.

- **Primary** — saturated background, white/black text. The page's main call to action.
- **Secondary** — pastel background + matching border. The everyday button.
- **Tertiary** — background-less at rest, fills on hover. Useful in dense toolbars.
- **Quaternary** — text only. No background, no border, no transitions.

For a one-off custom color, pass any literal CSS `<color>` value as the
`tone` — e.g. `tone="#ff1493"`, `tone="oklch(0.6 0.25 30)"`,
`tone="rgb(255 20 147)"`, or even a named color. The hue is extracted from
the input and run through the brand L/C curve, so every priority × state
slot (rest, hover, active, secondary alpha overlay, tertiary fill, …) is
filled automatically — no extra wiring required. Hover and active darken
on schedule; loading shimmer, focus ring, and disabled greys all keep
working because they're tone-agnostic.

Only the **hue** of the input is honoured. Lightness and chroma come from
the brand tone, so pale or low-chroma inputs (e.g. `tone="#cccccc"`) still
produce a legible button — saturation-matched to the rest of the
design system. If you need pixel-for-pixel control, set
`--button-fg-color` / `--button-bg-color` / `--button-br-color` directly
on `style`; inline-style declarations beat the resolver.

Dark mode reuses the same curve — a custom tone keeps the same color in
both modes. To pair light / dark variants, scope them yourself:
`.dark a-button[tone="…"] { --button-tone-source: …; }`.

## Paddingless

`paddingless` drops the button's outer padding to zero so it sits flush with
surrounding content. Useful for inline-link-style controls inside prose
("…or [discard the change]"). Restricted to `priority="quaternary"` per the
Figma spec — other priorities rely on the padding for their chip / pill shape.

## Underline (background-less priorities)

`underline` takes one of three style values:

- `underline="solid"`
- `underline="dashed"`
- `underline="dotted"`

Available on the two priorities that ship without a background fill —
`tertiary` and `quaternary`. Passing it on `primary` or `secondary` is a no-op
since the underline would clash with the filled chip. Colour inherits via
`currentColor`, so a tone shift updates the underline along with the label.

The treatment is intentionally subtler than the global `<a>` rule. The
underline holds at a 0.5px hairline in every state and only the alpha lifts on
interaction — the button already shifts its color (and on tertiary, its
background) on hover, so a heavier underline snap on top reads as a double
cue. Hover and active share the same state so clicking doesn't kick the
underline back to rest mid-press:

| State            | Thickness | Underline color                                    |
|------------------|-----------|----------------------------------------------------|
| Rest             | 0.5px     | 75% alpha of `currentColor` (mixed in oklch)       |
| Hover / active   | 0.5px     | `--button-fg-color` (priority/tone-aware, full alpha) |

Like the global `a:hover` rule, the hover underline color is set to the
resolved fg token directly (rather than `currentColor`) so the underline
repaints in lockstep with the label whenever the tone resolver flips
`--button-fg-color`.

## Children alongside label

The component renders `children` after the `label` and before the trailing
icon, so you can mix custom inline content — keyboard hints, "NEW" badges,
counters — with the standard prop-driven API. Children also work without
`label` set, if you want full control over the inner content.

```tsx
<Button label="Save" leadingIcon="check" tone="brand">
  <span style={{ opacity: 0.7, fontSize: 12 }}>⌘S</span>
</Button>
```

## Form submission

Setting `type="submit"` on a button inside (or associated with via
`form="id"`) a `<form>` triggers `form.requestSubmit()` on click. The
element additionally dispatches a custom `submitdetailed` event on the
form with `{ formData, submitter: { tag, attrs } }` in `detail`,
which is useful for analytics or wiring up multi-button forms.

`type="reset"` calls `form.reset()`.

## Custom click events

Adding `data-custom-event="some-name"` makes the button dispatch a
bubbling `CustomEvent("some-name")` on click — handy for instrumenting
analytics without taking ownership of `onClick`.

## Anchor mode

Setting `href` switches the rendered tag from `<a-button>` to
`<a role="button">`. Styles are identical (both selectors share the
CSS rule set). The form/submit/reset behaviour is no-op in anchor mode
because navigation takes over.

## Loading

`loading={true}` adds an animated diagonal-stripe overlay that slides
left → right across the button. The stripe colour is `currentColor`, so
it always tracks the label / icon tone without any per-tone token. It is
**visual only** — clicks still fire. Pair with `disabled` when the
action should be temporarily inert.

## Props

| Prop | Type | Notes |
|---|---|---|
| `tone` | `'brand' \| 'neutral' \| 'critical' \| 'info' \| 'success' \| 'warning' \| <CSS color>` | Default `brand`. Any literal CSS color (`'#ff1493'`, `'oklch(...)'`, ...) is accepted as a one-off custom tone — hue is extracted and run through the brand L/C curve. |
| `priority` | `'primary' \| 'secondary' \| 'tertiary' \| 'quaternary'` | Default `primary`. |
| `size` | `'small' \| 'default' \| 'large'` | Heights 24 / 28 / 32 — only padding changes. |
| `underline` | `'solid' \| 'dashed' \| 'dotted'` | **Tertiary / quaternary only** — no-op on filled priorities. |
| `paddingless` | `boolean` | **Quaternary only** — zeros outer padding for flush inline use. |
| `loading` | `boolean` | Stripe overlay; doesn't block clicks. Pair with `disabled` to inert. |
| `disabled` | `boolean` | Locks colours to the disabled palette + `pointer-events: none`. |
| `selected` | `boolean` | Toggled-on visual; shares the active-state look. |
| `leadingIcon` / `trailingIcon` | `IconShape` | Icon name from `a-icon.shapes.ts`. |
| `iconButton` | `boolean \| IconShape` | Collapses to a square + pins a min-width to the natural square. Pass a string (`iconButton="check"`) to set the icon in the same prop. |
| `label` | `string` | Rendered inside `<a-button-label>`. |
| `href` / `target` / `rel` | `string` | When `href` is set, renders `<a role="button">`. |
| `type` / `form` | `string` | Form `submit` / `reset` semantics. |

## Component tokens

The button exposes a set of CSS custom properties on its host element.
Override any of them on a single instance (`style={{ '--button-padding-x': '12px' }}`)
or on a selector that wraps the button (e.g. a tone priority of your own).
All colour tokens are dual-declared `hex; oklch`; alpha fills use
`color-mix(in oklch, …)` with a hex8 fallback.

| Token | Description |
|-------|-------------|
| `--button-fg-color` | Text and icon color (resolves via tone × priority × state). |
| `--button-bg-color` | Background fill. |
| `--button-br-color` | Border color. On primary buttons the resolver sets this to the fill so the 1px border merges with the bg; on secondary / tertiary / quaternary it's `transparent` (an alpha-matched border would stack on top of the alpha fill and read as a ring). |
| `--button-focus-color` | Focus-ring outline color. |
| `--button-padding-x`, `--button-padding-y` | Inner spacing. |
| `--button-br-radius` | Border radius. |
| `--button-br-width` | Border width. Default **`1px`**. Set to `0` to drop the border and tighten the bounding box by 2px in each dimension. |
| `--button-font-size`, `--button-font-family`, `--button-font-weight`, `--button-line-height` | Type. |
| `--button-gap` | Spacing between icon, label, and other children. |
| `--button-timing-in` / `--button-timing-out` / `--button-timing-active` | Per-state transition durations (`50ms`, `150ms`, `50ms` by default). Quaternary zeroes all three. |
| `--button-outline-offset` | Focus-ring offset from the edge (negative for inset). |
| `--button-loading-duration` | One slide cycle for the loading stripe (default `1s`). |
| `--button-loading-angle` | Stripe angle (default `125deg`). |
| `--button-loading-period` | Stripe + gap along the gradient axis (default `30px`). |
| `--button-loading-stripe`, `--button-loading-stripe-gap` | Stripe geometry (default `14px` / `16px`). |
| `--button-loading-opacity` | Overlay opacity (default `0.18` light, `0.22` dark). |
| `--button-loading-blur` | Optional softening of the stripe edges (default `0`). |
