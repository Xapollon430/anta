# Button

An action control. Renders an `<a-button>` web component (or
`<a role="button">` when `href` is set) with a tone Ã— priority matrix
applied via attributes.

```tsx

<Button tone="rebeccapurple" label="Named color" />
```

For custom tones, only the *hue* of the input is taken. Lightness and
chroma come from the brand-tone curve, so pale or low-chroma inputs
(`tone="#cccccc"`, `tone="white"`) still produce a legible button. The
full priority Ã— state matrix â€” rest, hover, active, secondary alpha
overlay, tertiary fill â€” is derived automatically.

If you need pixel-precise color, set `--button-fg-color`,
`--button-bg-color`, or `--button-br-color` directly via `style` â€”
inline declarations beat the resolver.

Dark mode reuses the same curve. To pair light/dark variants for a
custom tone, scope it yourself:

```css
.dark a-button[tone="â€¦"] { --button-tone-source: â€¦; }
```

## Priorities

Four priorities, each with a distinct visual signature:

| Priority | Visual |
|---|---|
| `primary` | Saturated fill. Main call to action. |
| `secondary` | Pastel background, no border. Everyday button. |
| `tertiary` | Background-less at rest, fills on hover. Dense toolbars. |
| `quaternary` | Text only â€” no background, no border, no transitions. |

<Preview>
  <Button tone="brand" priority="primary" label="Primary" />
  <Button tone="brand" priority="secondary" label="Secondary" />
  <Button tone="brand" priority="tertiary" label="Tertiary" />
  <Button tone="brand" priority="quaternary" label="Quaternary" />
</Preview>

```tsx
<Button tone="brand" priority="primary"    label="Primary" />
<Button tone="brand" priority="secondary"  label="Secondary" />
<Button tone="brand" priority="tertiary"   label="Tertiary" />
<Button tone="brand" priority="quaternary" label="Quaternary" />
```

Two prop constraints follow from priority and are enforced at the
type level â€” `underline` and `paddingless`.

### Underline

Only valid on `priority="tertiary"` / `"quaternary"`. Filled chips
don't need an underline; passing it on `primary` / `secondary` is a
TypeScript error. Three styles â€” `solid`, `dashed`, `dotted`:

<Preview>
  <Button tone="brand" priority="tertiary" underline="solid" label="Solid" />
  <Button tone="brand" priority="tertiary" underline="dashed" label="Dashed" />
  <Button tone="brand" priority="tertiary" underline="dotted" label="Dotted" />
</Preview>

```tsx
<Button tone="brand" priority="tertiary" underline="solid"  label="Solid" />
<Button tone="brand" priority="tertiary" underline="dashed" label="Dashed" />
<Button tone="brand" priority="tertiary" underline="dotted" label="Dotted" />
```

The underline mirrors the prose-link rule used by `<Text>`:

| State | Thickness | Alpha |
|---|---|---|
| Rest | 0.5px | 75% (`currentColor` mixed with transparent) |
| Hover / selected | 1px | 100% (`--button-fg-color`) |
| Active | 0.5px | 75% (returns to rest) |

The color tracks `--button-fg-color` so a tone shift updates the
underline in lockstep with the label.

### Paddingless

Only valid on `priority="quaternary"`. Zeros the outer padding so the
button sits flush with surrounding prose â€” useful when you want a
button to read as an inline link inside a sentence:

     promotes any text-only line inside JSX to its own `<p>`, which
     would split the button out of the surrounding paragraph and
     break inline flow. */}


## Icon-only

Pass `icon` alone â€” no `label`, no `trailingIcon`, no `children` â€” and
the button collapses to a square. No opt-in attribute: the CSS detects
icon-only mode structurally via `:has(> a-icon:only-child)`, so the
square padding + min-size pin apply automatically.

<Preview>
  <Button tone="brand" icon="dots-vertical" size="small" />
  <Button tone="critical" icon="trash" />
  <Button tone="brand" icon="check" size="large" />
</Preview>

```tsx
<Button tone="brand"    icon="dots-vertical" size="small" />
<Button tone="critical" icon="trash" />
<Button tone="brand"    icon="check"          size="large" />
```

A min-width is pinned to the natural square (20px / 24px / 28px for
small / default / large) so a tight flex parent can't clip the icon.

## States

<Preview>
  <Button tone="brand" loading label="Submitting" />
  <Button tone="brand" disabled label="Locked" />
  <Button tone="brand" selected label="Toggled on" />
  <Button tone="brand" loading disabled label="Critical" />
</Preview>

```tsx
<Button tone="brand" loading label="Submitting" />
<Button tone="brand" disabled label="Locked" />
<Button tone="brand" selected label="Toggled on" />
<Button tone="brand" loading disabled label="Critical" />
```

- **`loading`** â€” diagonal stripe overlay slides across the button.
  Stripe color follows `currentColor`, so it tracks the tone. Blocks
  clicks via `pointer-events: none`.
- **`disabled`** â€” locks the colors to the disabled palette, sets
  `pointer-events: none`, and removes the button from the tab order.
  Beats inline `--button-bg-color` overrides.
- **`selected`** â€” toggled-on / pressed visual; shares the active
  state's look. Useful for filter chips and icon toggles.

## Anchor mode

Setting `href` switches the rendered tag from `<a-button>` to
`<a role="button">`. Styling is identical â€” both selectors share the
same CSS rules. Form-submission props (`type`, `form`) are TypeScript
errors in anchor mode; navigation takes over.

<Preview>
  <Button tone="brand" href="#" rel="noopener" label="Read the docs" trailingIcon="external-link" />
</Preview>

```tsx
<Button tone="brand" href="/docs" target="_blank" rel="noopener" label="Read the docs" />
```

## Form submission

For non-anchor buttons, `type="submit"` and `type="reset"` integrate
with native forms:

<Preview>
  {/* `action="javascript:void(0)"` keeps the live demo from navigating
       on submit; in real code you'd give the form a real action or
       intercept via JS. */}
  <form id="signup-demo" action="javascript:void(0)" style={{ display: 'flex', gap: '8px' }}>
    <Button tone="brand" type="submit" label="Sign up" />
    <Button tone="brand" type="reset" label="Clear" />
  </form>
  <Button tone="brand" type="submit" form="signup-demo" label="Submit from outside" />
</Preview>

```tsx
<form id="signup">
  <Button tone="brand" type="submit" label="Sign up" />
  <Button tone="brand" type="reset"  label="Clear" />
</form>

{/* Associate with a form by id when the button is outside */}
<Button tone="brand" type="submit" form="signup" label="Submit from outside" />
```

`type="submit"` calls `form.requestSubmit()` and also dispatches a
`submitdetailed` event on the form with `{ formData, submitter: { tag, attrs } }`
in `detail` â€” handy for analytics or multi-button forms.

## Custom click events

`data-custom-event="<name>"` makes the button dispatch a bubbling
`CustomEvent("<name>")` on click. Use it to instrument analytics
without taking ownership of `onClick`:

<Preview>
  <Button tone="brand" label="Save" data-custom-event="save-clicked" />
</Preview>

```tsx
<Button tone="brand" label="Save" data-custom-event="save-clicked" />
```

## Props

| Prop | Type | Notes |
|---|---|---|
| `tone` | `'neutral' \| 'brand' \| 'critical' \| 'info' \| 'success' \| 'warning' \| <css color>` | Default `neutral`. Literal colors derive the full matrix from the input hue. |
| `priority` | `'primary' \| 'secondary' \| 'tertiary' \| 'quaternary'` | Default `primary`. |
| `size` | `'small' \| 'default' \| 'large'` | Default `default`. Only padding changes. |
| `underline` | `'solid' \| 'dashed' \| 'dotted'` | `tertiary` / `quaternary` only. |
| `paddingless` | `boolean` | `quaternary` only. |
| `loading` | `boolean` | Stripe overlay; blocks clicks. |
| `disabled` | `boolean` | Disabled palette; no clicks; removed from tab order. |
| `selected` | `boolean` | Toggled-on visual (shares active look). |
| `icon` / `trailingIcon` | `IconShape` | Icon shape names. When `icon` is set alone (no `label`, no `trailingIcon`, no `children`) the CSS auto-collapses the button to a square via `:has(> a-icon:only-child)`. |
| `label` | `string` | Rendered inside `<a-button-label>`. |
| `href` | `string` | Switches to `<a role="button">`. |
| `target` / `rel` | `string` | Anchor mode only. |
| `type` | `'button' \| 'submit' \| 'reset'` | Form integration. Forbidden with `href`. |
| `form` | `string` | Associate with a form by id. Forbidden with `href`. |
| `onClick` | `(e) => void` | Click handler. |
| `data-custom-event` | `string` | Dispatches `CustomEvent(<name>)` on click. |

## Component tokens

Override any of these on a single instance
(`style={{ '--button-padding-x': '12px' }}`) or on a selector wrapping
the button (e.g. a tone or variant of your own). Color tokens are
dual-declared `hex; oklch`; alpha fills use `color-mix(in oklch, â€¦)`
with a hex8 fallback first.

| Token | Description |
|---|---|
| `--button-fg-color` | Text and icon color. Resolved per tone Ã— priority Ã— state. |
| `--button-bg-color` | Background fill. |
| `--button-br-color` | Border color. Primary mirrors the fill; others are transparent. |
| `--button-focus-color` | Focus-ring outline color. |
| `--button-padding-x` / `--button-padding-y` | Inner spacing. |
| `--button-br-radius` | Border radius (default `4px`). |
| `--button-br-width` | Border width (default `0` â€” no border). Set to `1px` for an outlined variant. |
| `--button-font-size` / `--button-font-family` / `--button-font-weight` / `--button-line-height` | Type. |
| `--button-gap` | Spacing between icon, label, and children. |
| `--button-timing-in` / `--button-timing-out` / `--button-timing-active` | Per-state transition durations (`50ms` / `150ms` / `50ms`). Quaternary zeroes all three. |
| `--button-outline-offset` | Focus-ring offset (default `1px`). |
| `--button-loading-duration` | One slide cycle for the loading stripe (default `1.1s`). |
| `--button-loading-angle` | Stripe angle (default `125deg`). |
| `--button-loading-period` | Stripe + gap along the gradient axis (default `30px`). |
| `--button-loading-stripe` / `--button-loading-stripe-gap` | Stripe geometry (default `14px` / `16px`). |
| `--button-loading-opacity` | Overlay opacity (default `0.3`). |
| `--button-loading-blur` | Stripe-edge softening (default `1px`). |
