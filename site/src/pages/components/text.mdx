# Text

A block-level text container that scopes the color hierarchy of its
contents. Use `priority` to dial the emphasis from `primary` (most
prominent, `--text-1`) down to `quinary` (least prominent, `--text-5`);
the **default is `secondary` (`--text-2`)** â€” body text reads a step
softer than the strongest foreground, so reach for `primary` when you
want emphasis. Pair with `tone` to apply a tinted palette (`brand`,
`success`, `critical`, `warning`, `info`). Pass `inline` to render as
`inline-block` for use in the middle of other text.

`Text` also owns Anta's prose-link styling. Links nested inside
`<Text>` are styled by the design system â€” there are no per-page
overrides to maintain.

## Link behavior

Links inside `<Text>` follow a priority-aware hierarchy that keeps
emphasis consistent with the surrounding text:

| Priority      | Link color (default)              | Link color (hover)            |
|---------------|-----------------------------------|-------------------------------|
| `primary`     | `--link-color` (brand blue)       | `--link-color-hover`          |
| `secondary`   | `--link-color`                    | `--link-color-hover`          |
| `tertiary`    | `currentColor` (= `--text-3`)     | `--text-2`                    |
| `quaternary`  | `currentColor` (= `--text-4`)     | `--text-3`                    |
| `quinary`     | `currentColor` (= `--text-5`)     | `--text-4`                    |

The underline is a 0.5px hairline at 75% alpha by default and bumps to
1px / 100% alpha on hover. On `:active` it returns to the 0.5px / 75%
resting state.

For tinted text (`tone="brand|success|critical|warning|info"`) links
are **always** muted regardless of priority â€” they take `currentColor`
and step up to the next-stronger level of the same tint on hover. At
`priority="primary"` there is no level above, so hovering only brings
the underline to full alpha.

## Truncation & expansion

<Col>

<TruncateDemo client:load />

`truncate` accepts either `true` (or `1`) for a single-line ellipsis or
any integer â‰Ą 2 for multi-line clamping; `0` or a negative value means
no truncation. Both modes use `-webkit-line-clamp` inside
`display: -webkit-box`. Despite the `-webkit-` prefix, every major
browser supports it (Firefox 68+, Chrome, Safari, Edge). The host gets
`min-width: 0` so truncation works correctly inside flex and grid
containers.

```tsx folded
{/* Single line */}
<Text truncate>{longSentence}</Text>

{/* Multi-line â€” pass the line count */}
<Text truncate={2}>{longSentence}</Text>
<Text truncate={3}>{longSentence}</Text>
```

</Col>

<ExpandableDemo client:load />

Pair `expandable` with `truncate` to let the user reveal the full
text. Single-line truncation gets a horizontal fade on the right edge;
multi-line gets a vertical fade over the last line. A chevron-down
hint appears in the bottom-right corner on hover or keyboard focus.
The chevron is a real `<button>` carrying `aria-expanded` â€” clicking it
(or focusing it and pressing Enter) drops the clamp and reveals the
full text. `expandable` only takes effect together with `truncate`.

By default expanding is **one-way** â€” once revealed, the control is
removed. Add `collapsible` to make it a two-way toggle: the chevron
stays visible while expanded (rotated to point up) and flips between
"Show more" and "Show less". `collapsible` only takes effect together
with `expandable`.

```tsx folded
<Text truncate expandable>{longSentence}</Text>
<Text truncate={3} expandable>{longSentence}</Text>

{/* Two-way: reader can collapse it back */}
<Text truncate={5} expandable collapsible>{longSentence}</Text>
```

<Disclosure title="Styling" open>

Colour comes from **`priority`** (the neutral text level) and **`tone`** (the tint);
`size` and the type props set the rest. `Text` doesn't expose its own tokens â€” it
consumes the global **text role tokens** (`--text-1` â€¦ `--text-5`, the `-{tone}`
tints, `--link-color`). Retune those once on `:root` / `.dark` to shift text colour
everywhere â€” see [Colors](/colors/).

```tsx folded
<Text priority="secondary" tone="brand">Tinted secondary text</Text>
```

`<a-text>` is light-DOM, so one-offs are plain CSS on the element or its prose links:

```css folded
a-text a { text-decoration-thickness: 2px; }
```

</Disclosure>
