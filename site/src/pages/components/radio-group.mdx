# RadioGroup

The single-select container. `RadioGroup` renders an `<a-radio-group>` web
component that coordinates a set of [`Radio`](/components/radio/) options: exactly
one is selected at a time, arrow keys move the selection, and the group is a
form-associated element, so a wrapping `<form>` submits one `name=value`.

## Controlled vs. uncontrolled

<Columns>
<Col>

Like the rest of anta's stateful elements, the group works either way.
**Uncontrolled** — pass `defaultValue` and let the group own selection:

```tsx folded
<RadioGroup name="plan" defaultValue="pro">
  <Radio value="free">Free</Radio>
  <Radio value="pro">Pro</Radio>
  <Radio value="team">Team</Radio>
</RadioGroup>
```

</Col>
<Col>

**Controlled** — pass `value` + `onStateChange` and own it yourself. The group
fires *before* applying, so you accept by updating `value` and reject by doing
nothing (or `event.preventDefault()` in the uncontrolled case):

```tsx folded
const [plan, setPlan] = useState('pro')

<RadioGroup name="plan" value={plan} onStateChange={(e, { next }) => setPlan(next)}>
  <Radio value="free">Free</Radio>
  <Radio value="pro">Pro</Radio>
  <Radio value="team">Team</Radio>
</RadioGroup>
```

</Col>
</Columns>

<Col>

Six named tones — `brand` (default), `neutral`, `info`, `success`, `warning`,
`critical`. Omitting `tone` resolves to `brand`. In addition, a custom tone is
possible: pass any literal CSS colour and the group will adapt to it. Set it on
the `RadioGroup` to apply to every option, or on a single
[`Radio`](/components/radio/) to override just that one.

## Keyboard & accessibility

The group is one tab stop: `Tab` moves into the selected option (or the first
one), and the arrow keys (`↑`/`↓`/`←`/`→`) move the selection between enabled
options, wrapping at the ends. `Space` selects the focused option.

The `RadioGroup` wrapper sets `role="radiogroup"` on the group and `role="radio"`
on every option; the group then stamps `aria-checked` (`"true"` / `"false"`)
on each radio to mirror the live selection — including arrow-key navigation
in uncontrolled mode. Follows the
[WAI-ARIA radio-group pattern](https://www.w3.org/WAI/ARIA/apg/patterns/radio/).

## Forms

`<a-radio-group>` is a form-associated custom element. Give it a `name` and the
selected option's `value` is submitted with the form like a native radio group:

```tsx folded
<form>
  <RadioGroup name="contact" defaultValue="email">
    <Radio value="email">Email</Radio>
    <Radio value="sms">SMS</Radio>
  </RadioGroup>
</form>
// FormData → contact=email
```
