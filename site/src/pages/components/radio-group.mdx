# RadioGroup

The single-select container. `RadioGroup` renders an `<a-radio-group>` web
component that coordinates a set of [`Radio`](/components/radio/) options: exactly
one is selected at a time, arrow keys move the selection, and the group is a
form-associated element, so a wrapping `<form>` submits one `name=value`.

All the state lives in the element, so the behaviour is identical whether you use
the wrapper or author the `<a-radio-group>` tag by hand.

```tsx folded
<RadioGroup
  label="Select an option"
  hint="Choose how we should reach you."
  defaultValue="email"
>
  <Radio value="email">Email</Radio>
  <Radio value="sms">SMS</Radio>
  <Radio value="push">Push notification</Radio>
</RadioGroup>
```

</Col>
<Col>

`label` prints above the options; `hint` prints below — both plain text passed
as attributes. The element renders them through named `<a-radio-label>` and
`<a-radio-hint>` sub-elements inside its shadow, so hand-authored HTML uses the
same attributes:

```html folded
<a-radio-group name="contact" label="Select an option"
               hint="Choose how we should reach you.">
  <a-radio value="email">Email</a-radio>
  <a-radio value="sms">SMS</a-radio>
</a-radio-group>
```

</Col>
</Columns>

## Controlled vs. uncontrolled

<Columns>
<Col>

Like the rest of anta's stateful elements, the group works either way.
**Uncontrolled** — pass `defaultValue` and let the group own selection:

```tsx folded
<RadioGroup name="plan" defaultValue="pro">
  <Radio value="free">Free</Radio>
  <Radio value="pro">Pro</Radio>
  <Radio value="team">Team</Radio>
</RadioGroup>
```

</Col>
<Col>

**Controlled** — pass `value` + `onChange` and own it yourself. The group then
only *requests* a change, so you can reject one by not updating `value`:

```tsx folded
const [plan, setPlan] = useState('pro')

<RadioGroup name="plan" value={plan} onChange={setPlan}>
  <Radio value="free">Free</Radio>
  <Radio value="pro">Pro</Radio>
  <Radio value="team">Team</Radio>
</RadioGroup>
```

</Col>
</Columns>

<Col>

Disable the entire set with `disabled` on the `RadioGroup`. To disable a single
option instead, put `disabled` on the [`Radio`](/components/radio/). Disabled
options are skipped by keyboard navigation.

## Keyboard & accessibility

The group is one tab stop: `Tab` moves into the selected option (or the first
one), and the arrow keys (`↑`/`↓`/`←`/`→`) move the selection between enabled
options, wrapping at the ends. `Space` selects the focused option.

> **Note:** ARIA roles (`role="radiogroup"` / `role="radio"` / `aria-checked`)
> are not wired yet — they land in a later accessibility pass. Keyboard
> operation and form submission already work.

## Forms

`<a-radio-group>` is a form-associated custom element. Give it a `name` and the
selected option's `value` is submitted with the form like a native radio group:

```tsx folded
<form>
  <RadioGroup name="contact" defaultValue="email">
    <Radio value="email">Email</Radio>
    <Radio value="sms">SMS</Radio>
  </RadioGroup>
</form>
// FormData → contact=email
```
