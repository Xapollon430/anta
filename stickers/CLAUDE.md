# @antadesign/stickers — Antithesis sticker pack

A companion package to `@antadesign/anta`, published separately on npm so that
core anta carries **no `lottie-web`** and no sticker code. Apps that don't use
stickers never install the animation runtime; apps that do add one package
(`@antadesign/stickers`), which pulls `@antadesign/anta` + `lottie-web`
transitively.

This package follows all the root [`../CLAUDE.md`](../CLAUDE.md) conventions
(two tiers — `src/elements/` web components + JSX wrappers; declarative DOM;
boolean-attribute presence matching; `color-mix(in oklch, …)`; docs-in-sync).
Only the package-specific notes live here.

## Layout

- `src/elements/a-sticker.{ts,css}` — static SVG carrier `<a-sticker>`.
- `src/elements/a-sticker-animated.{ts,css}` — Lottie carrier `<a-sticker-animated>`. **The only `lottie-web` importer.**
- `src/elements/index.ts` — barrel registering both elements.
- `src/Sticker.tsx`, `src/StickerAnimated.tsx` — JSX wrappers. They emit `<a-sticker>` / `<a-sticker-animated>` and import shared helpers from `@antadesign/anta` (`@antadesign/anta/anta_helpers`, `@antadesign/anta/general_types`).
- `src/jsx.d.ts` — augments `@antadesign/anta/jsx-runtime`'s `JSX.IntrinsicElements` with the two sticker tags (anta itself ships no sticker types).
- `src/assets/<name>/<name>.{json,svg}` — source artwork (Lottie JSON + static SVG pose). One subfolder per sticker.
- `src/generated/` — **generated, committed** per-sticker modules + barrel. Do not hand-edit; run `pnpm run stickers` after changing `src/assets/`.
- `scripts/generate-stickers.mjs` — the generator (also published for consumers to build their own packs; `--package` sets the emitted import specifier).

## Coupling to anta

The JSX runtime is anta's (`jsxImportSource: "@antadesign/anta"`), and `@antadesign/anta` is a **regular dependency** (`workspace:*` in-repo). `lottie-web` is this package's own dependency; `react` is a peer.

## Build & dev

Same shape as anta: `pnpm run build` = `build:js` (esbuild) + `build:css` (copy CSS + the generator) + `build:types` (tsc). `dist/` is gitignored and regenerated by `prepare` / `prepublishOnly`. The repo-root `pnpm run dev` builds this package after anta and before the site.

```sh
pnpm run stickers   # regenerate src/generated/ from src/assets/
pnpm run build      # full build
pnpm run typecheck
```

## Publishing

See the **npm publishing** section in the [root `../CLAUDE.md`](../CLAUDE.md) — it covers both packages and the order that matters. The two package-specific things to remember here:

- **Publish `@antadesign/anta` first.** This package's `workspace:*` dependency on anta is rewritten to anta's exact current version at pack time, so that anta version must already be on npm (and carry the exports this package imports — `./anta_helpers`, `./general_types`).
- **Use `pnpm publish`, not `npm publish`** — only pnpm rewrites the `workspace:*` protocol. Access + tag come from `publishConfig` (`{ access: "public", tag: "dev" }`), so no flags needed:

```sh
npm version prerelease --preid=dev   # bump first (immutable versions)
pnpm publish --no-git-checks
```
